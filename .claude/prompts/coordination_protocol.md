# Multi-Agent Coordination Protocol

## Agent Activation and Task Distribution

### Primary Agent Responsibilities Matrix

| Task Type | Primary Agent | Supporting Agents | Coordination Level |
|-----------|---------------|-------------------|-------------------|
| SAOM Model Development | SAOM Specialist | Statistical Analyst, Code QA | High |
| Custom C++ Effects | SAOM Specialist | Code QA, Simulation Engineer | Critical |
| Statistical Analysis | Statistical Analyst | Research Methodologist | High |
| Theoretical Framework | Tolerance Theory Expert | Statistical Analyst, Research Methodologist | Medium |
| Simulation Infrastructure | Simulation Engineer | Code QA, SAOM Specialist | High |
| Quality Assurance | Code QA | All Agents | Continuous |
| Research Standards | Research Methodologist | All Agents | Oversight |

### Communication Protocols

#### Critical Decision Points
1. **Model Specification Changes**: Requires consensus from SAOM Specialist, Statistical Analyst, and Tolerance Theory Expert
2. **Effect Size Interpretation**: Joint decision by Statistical Analyst and Tolerance Theory Expert
3. **Computational Approach**: Agreement between SAOM Specialist and Simulation Engineer
4. **Quality Standards**: Code QA and Research Methodologist approval required

#### Information Sharing Requirements
- **Daily Sync**: Brief status updates on current work
- **Weekly Review**: Comprehensive progress assessment and issue resolution
- **Milestone Gates**: Formal review before proceeding to next phase
- **Documentation Updates**: Real-time sharing of discoveries and modifications

## Collaborative Workflows

### Phase 1: Theoretical Development and Model Specification
```
Lead: Tolerance Theory Expert + SAOM Specialist
Support: Statistical Analyst, Research Methodologist

Activities:
1. Finalize micro-theory specifications
2. Design SAOM effect structure
3. Identify custom C++ requirements
4. Establish statistical validation criteria

Deliverable: Complete theoretical framework with implementable SAOM specification
```

### Phase 2: Implementation and Testing
```
Lead: SAOM Specialist + Simulation Engineer
Support: Code QA, Statistical Analyst

Activities:
1. Develop custom RSiena C++ effects
2. Implement simulation infrastructure
3. Create testing frameworks
4. Validate computational correctness

Deliverable: Working SAOM implementation with custom effects
```

### Phase 3: Empirical Calibration and Validation
```
Lead: Statistical Analyst + SAOM Specialist
Support: Research Methodologist, Tolerance Theory Expert

Activities:
1. Estimate models on empirical data
2. Assess model fit and convergence
3. Validate theoretical predictions
4. Document methodological approach

Deliverable: Empirically calibrated SAOM parameters
```

### Phase 4: Simulation Experiments
```
Lead: Simulation Engineer + Statistical Analyst
Support: SAOM Specialist, Code QA

Activities:
1. Execute parameter sweep experiments
2. Monitor computational performance
3. Validate simulation outputs
4. Compile results for analysis

Deliverable: Complete simulation experiment results
```

### Phase 5: Analysis and Documentation
```
Lead: Statistical Analyst + Research Methodologist
Support: All Agents

Activities:
1. Comprehensive statistical analysis
2. Theoretical interpretation of findings
3. Academic documentation preparation
4. Reproducibility verification

Deliverable: Publication-ready research outputs
```

## Quality Gates and Review Checkpoints

### Technical Review Gates
1. **Custom Effect Validation**: Code QA + SAOM Specialist
2. **Statistical Method Approval**: Statistical Analyst + Research Methodologist
3. **Theoretical Coherence Check**: Tolerance Theory Expert + Research Methodologist
4. **Performance Validation**: Simulation Engineer + Code QA
5. **Reproducibility Verification**: Research Methodologist + Code QA

### Academic Standards Checkpoints
- **Literature Integration**: Tolerance Theory Expert validation
- **Methodological Rigor**: Research Methodologist approval
- **Statistical Validity**: Statistical Analyst certification
- **Technical Implementation**: SAOM Specialist + Code QA verification
- **Computational Efficiency**: Simulation Engineer benchmarking

## Conflict Resolution and Decision Making

### Technical Disagreements
1. **Issue Identification**: Clear articulation of the technical problem
2. **Expert Consultation**: Relevant agent expertise identification
3. **Alternative Evaluation**: Systematic comparison of approaches
4. **Consensus Building**: Collaborative decision with documented rationale
5. **Implementation Validation**: Post-decision verification of outcomes

### Methodological Disputes
1. **Literature Review**: Comprehensive assessment of established practices
2. **Expert Opinion**: Input from Research Methodologist and domain experts
3. **Pilot Testing**: Small-scale validation of competing approaches
4. **Academic Consultation**: External expert input when needed
5. **Documentation**: Complete rationale recording for transparency

## Communication Standards

### Technical Documentation
- **Precision**: Exact specification of methods and parameters
- **Completeness**: All necessary details for replication
- **Clarity**: Accessible to both technical and non-technical team members
- **Currency**: Real-time updates as work progresses

### Progress Reporting
```
Daily Update Format:
- Current task focus
- Progress made
- Obstacles encountered
- Assistance needed
- Next 24-hour priorities

Weekly Review Format:
- Major accomplishments
- Critical decisions made
- Quality assurance status
- Timeline adherence
- Resource requirements
```

### Academic Communication
- **Formal Terminology**: Consistent use of discipline-specific language
- **Evidence-Based**: All claims supported by data or literature
- **Uncertainty Acknowledgment**: Clear communication of limitations
- **Collaborative Tone**: Respectful and constructive interaction

## Emergency Protocols

### Critical Technical Failures
1. **Immediate Notification**: Alert all agents to critical issues
2. **Expert Mobilization**: Deploy relevant specialists immediately
3. **Alternative Planning**: Develop backup approaches
4. **Documentation**: Record problems and solutions for future reference

### Timeline Threats
1. **Risk Assessment**: Evaluate impact on overall project timeline
2. **Resource Reallocation**: Adjust agent assignments as needed
3. **Scope Adjustment**: Modify objectives if necessary to maintain quality
4. **Communication**: Transparent updates to all stakeholders

## Success Metrics

### Individual Agent Performance
- **Quality of Deliverables**: Technical excellence and academic rigor
- **Timeliness**: Adherence to agreed schedules
- **Collaboration**: Effective interaction with other agents
- **Innovation**: Creative solutions to technical challenges

### Team Performance
- **Integration**: Seamless collaboration across agents
- **Quality**: Overall research output excellence
- **Efficiency**: Effective use of time and resources
- **Impact**: Contribution to academic knowledge and practical applications
